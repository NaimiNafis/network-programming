--------------------------------------------------

コンパイルコマンド:
```
gcc -I../mynet -L../mynet -o task5 task5.c -lmynet -lpthread
```

コマンド:
```
./task5 [username] [port_number]
```

テストコマンド:
```
./idobata_test_mac
```


--------------------------------------------------

プログラムの特徴:

- **ノンブロッキングモード**:
  ソケットをノンブロッキングモードに設定することで、複数のクライアントを効率的に扱うことができる。
- **クライアント管理**:
  クライアントの配列を初期化して管理することで、接続の管理が容易になる。
- **メッセージブロードキャスト**:
  クライアントからのメッセージを処理し、他のクライアントにブロードキャストする機能を持っている。
- **デバッグ情報**:
  詳細なデバッグ情報を表示するには、コード内のデバッグステートメント（コメントアウトされているprintfステートメント）をコメント解除してください。

--------------------------------------------------

苦労した点:

- **クライアントの接続管理**:
  - クライアントの接続を管理するために、配列を使用してソケットディスクリプタを追跡する必要があった。
  - クライアントが接続を切断したときに、適切にソケットをクローズし、配列を更新する処理を実装するのが難しかった。
- **チャットメッセージのシームレスな表示**:
  - サーバーとクライアント間でメッセージの表示をシームレスにし、デバッグ情報が混在しないようにする調整が必要だった。

--------------------------------------------------

最終結果:

最終的に、サーバーとクライアントのコードを適切に修正し、すべてのテストに合格することができた。
これにより、複数のクライアントからの接続を正しく管理し、メッセージをブロードキャストするチャットサーバーを完成させることができた。
